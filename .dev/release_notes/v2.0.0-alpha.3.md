## Headline — `.auto` now defaults to JIT
- The default engine (`wasm_instance_new` / `Module.instantiate(.{})` / `zwasm run`) is now
  **JIT-preferring with interp fallback**. The gating blocker — **D-489/D-494**, an x86_64 JIT
  regalloc miscompile (LSRA dual spill-slot mint collision, unified on `n_spill_minted`) — is FIXED.
- **JIT-backed instances expose the full embedding C-API surface** (memory/table/global externs,
  introspection, `zwasm_instance_get_func`, accessors) — ADR-0200 / D-496.
- **funcref through the JIT C-API** (D-498): table-recovered + call-result funcref round-trip +
  are callable (`wasm_ref_as_func` by func_idx + funcref-result marshalling; closed a Win64 SEGV).
- CLI `--engine interp|jit` honoured on the captured-run path; `--fuel`/`--timeout` meter the JIT.

## Highlights since alpha.2

### Conformance — 100% spec
- Wasm **1.0 / 2.0 / 3.0** all green, **0 skips** (GC 362/0). `test-spec` 25539/0.
- Exception Handling on the interpreter **and JIT, both arches** (arm64 + x86_64).
- WASI **0.1** complete; **0.2 / Component Model** default-on (corpus 158/0/0);
  **0.3 core** done, incl. cross-component STRING/list/record composition (D-305).

### JIT engine
- JIT correct on **both arm64 and x86_64**; realworld corpus interp-vs-JIT
  byte-matches wasmtime (gating `test-realworld-diff-jit`, 56 fixtures).
- **v1-JIT performance parity** restored (D-265 loop-local regalloc rework closed).
- SIMD (v128) is JIT-backed and spill-complete (D-034 / D-460 / D-461 closed).

### Embedding APIs (the alpha.3 surface story)
- **ADR-0200 JIT-backed embedding API**: dual-engine accessors, `exportFuncSig`,
  export-types on JIT, FP/mixed multi-arg `--invoke` dispatch matrix.
- **C-API 293/293**, Windows-export complete; **Zig-API** complete with full
  WASI parity; lean CLI. Memory-safety sound (ReleaseSafe runners, ADR-0177).

### Robustness / hardening
- interp + JIT fuzzing **0 crashes**; exec-differential fuzz clean.
- v128-in-GC-type conformance class swept (typed `select`/`array.new_data`
  v128 fixed; v128 `array.fill`/`new` host-panic guarded → clean trap).

## Known limitations (debt-tracked, non-blocking — all interp-pinned so the DEFAULT never crashes)
- **D-497** funcref-table `grow` on a JIT instance (externref grow works); **D-498-residual** none
  (funcref table-call + result done); **D-499** x86_64 trivial-leaf-fn `fuel=0`/interrupt
  immediate-trap (runaway-safety intact — loops are back-edge-polled); **D-500** Win64 component
  CM-API string-arg wrapper-thunk → the component CM-API core is `.interp`-pinned (cljw consumes
  components on interp anyway). Component-on-JIT is a future capability.
- **Windows host-embedding examples**: an intermittent file-create (missing-parent-dir) ENV flake
  on the Win64 host examples — the test suites themselves are all green (`windows-host-example-filecreate`).
- D-492 (typed-ref `select`), D-495 (proper v128 `array.fill` value-marshal): ultra-niche, guarded.

## Migration
- v1 ABI compatibility is out of scope (ADR-0156); surfaces (C/Zig/CLI) are
